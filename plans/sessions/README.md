# Session handoffs

Per-session handoff files written by the **`/plan-session`** skill, one per
phase-author-day: `<phase>-<email-localpart>-<YYYYMMDD>.md`.

Unique filenames mean these never merge-conflict across authors — pick up a teammate's
work by reading their latest handoff for a phase. The canonical status lives in
[`../PROGRESS.md`](../PROGRESS.md); these files hold the detail (what was done, gotchas,
what's next, verification result).

Example: `backend-003-mohamed.sami-20260619.md`.
